## 🔶 核心原则

1. **所有结论必须有数据支撑**——观点不能凭空产生
2. **数据必须可追溯**——报告中每个数字都能追溯到对应的 Skill 输出
3. **表述必须中立**——不夸大、不淡化、不主观臆断

## 第一部分：数据准确性检查

### 1.0 原始数据转录检查（最重要）

生成报告中最常见的错误是 AI 将 Skill 返回的数据**填错**。

| 检查项 | 方法 | 示例错误 |
|--------|------|----------|
| PE/PB 值是否与 stock-analyzer 输出一致 | 逐项比对 | stock-analyzer 返回 PE=7.7，报告写成 7.2 |
| 营收/净利润是否与 financial-analysis 一致 | 逐行比对 | 财务数据填错年份 |
| ROCE 是否与 roce-calculator 一致 | 逐项比对 | roce-calculator 返回 19.12%，报告写成 17.5% |
| 股息率是否与 stock-analyzer 一致 | 比对 | 股息率 TTM 与静态混淆 |
| 分位数是否与 risk-analysis 一致 | 比对 | 3 年分位与 5 年分位混淆 |
| 涨跌幅与现价是否一致 | 验算 | 现价 26.89，涨跌幅 -1.21%，计算前后价格是否自洽 |

> **最易出错字段 TOP 3**：财务数据（年份错位）、ROCE（自行计算而非使用 skill 返回值）、分位数（周期混淆）

### 1.1 数据来源交叉验证

| 数据项 | 主要来源 | 交叉验证来源 |
|--------|----------|-------------|
| 现价/涨跌幅 | stock-analyzer（Tushare） | akshare `stock_zh_a_spot_em()` |
| PE/PB | stock-analyzer（Tushare Pro） | akshare 新浪接口 |
| 营收/净利润 | financial-analysis（新浪财经） | 财报 PDF 校验（如可用） |
| 分红 | a-dividend-analyzer（东方财富） | 公司公告 |
| ROCE | roce-calculator（新浪财报） | 自行验算：EBIT ÷（总资产 - 流动负债） |

**交叉验证触发条件**：
- 某个数据点偏离行业均值超过 3 个标准差
- 同比变化超过 100%（需确认是否有特殊原因）
- 数据与其他 Skill 返回的关联数据矛盾（如 ROE 很高但 ROCE 很低）

### 1.2 计算逻辑复核

报告中自行计算的关键指标必须验证：

| 指标 | 计算公式 | 检查要点 |
|------|----------|----------|
| 营收增长率 | （当年 - 上年）/ 上年 | 正负号是否正确，是否用对年份 |
| 净利润增长率 | 同上 | 同上 |
| ROCE | EBIT /（总资产 - 流动负债） | **必须使用 roce-calculator 返回值，不得自行计算** |
| 综合评分 | 6 维度求和 | 各子项是否在 0-20 分范围内 |
| 股息率 | 每股分红 / 股价 | 是否用 TTM 分红还是上一年度分红 |

### 1.3 单位和格式检查

| 检查项 | 正确格式 | 常见错误 |
|--------|----------|----------|
| 金额单位 | 亿（除非特别注明） | 万/亿混淆 |
| 百分比 | 保留 2 位小数 | 缺少 % 符号 |
| 股价精度 | 2 位小数 | 多余小数位 |
| 市值单位 | 亿 | 写成元 |
| 日期格式 | YYYY-MM-DD | 格式不统一 |
| 空白数据 | 显示为"-" | 写 0 或留空 |

### 1.4 时间一致性检查

| 检查项 | 要求 |
|--------|------|
| 报告日期与行情日期一致 | 所有数据的日期应相同 |
| 财务数据年度对齐 | 同一列数据必须同一年份 |
| 分位数计算周期标注 | 明确标注是 3月/1年/3年/5年 |
| 对比基准期一致 | 同比 vs 环比不能混淆 |

## 第二部分：评估观点客观性检查

### 2.0 观点是否有数据支撑

每个观点/结论必须能在报告数据中找到对应依据。

| ✅ 客观表述 | ❌ 主观表述 |
|------------|------------|
| "PE 为 7.7，低于行业均值 15，估值偏低" | "这只股票被严重低估" |
| "近 3 年营收增长 5%-8%，增速放缓" | "增长乏力，前景堪忧" |
| "ROCE 从 20% 降至 12%，下降 40%" | "盈利能力大幅恶化" |
| "十大股东集中度 62%，较上期下降 5 个百分点" | "股权结构分散，存在风险" |

**检查原则**：
- 每个"形容词"（低估/高估/优秀/差）后面必须跟数据
- 不得使用没有量化标准的绝对化表述（如"极具潜力"、"非常安全"）
- 不得使用情感色彩强烈的词汇（如"令人担忧"、"值得期待"）

### 2.1 风险评估客观性

| 检查项 | 说明 |
|--------|------|
| 风险等级是否与风险分析 skill 输出一致 | 不夸大、不缩小风险 |
| 每项风险是否有具体说明 | "经营风险"→ 需说明具体原因 |
| 正面/负面新闻权重是否均衡 | 不能只看正面或只看负面 |
| 是否区分"已发生风险"和"潜在风险" | 两者标注应不同 |

### 2.2 投资建议推导逻辑

投资建议必须展示完整的推导链条：

```
数据 → 分析 → 判断 → 建议
```

**检查流程**：
1. 建议是"买入/持有/卖出"还是"强烈买入/减仓/止损"
2. 该建议是基于哪个（些）维度得出的
3. 目标价的设定是否有依据（PE 估值法？DCF？可比公司？）
4. 是否考虑了反面情景（如果数据不准确怎么办）

**常见问题**：
| 问题 | 示例 | 修正 |
|------|------|------|
| 建议与评分矛盾 | 评分 D（较差），建议"逢低布局" | 评分差应匹配谨慎建议 |
| 目标价无依据 | "目标价 50 元"但不说明计算方法 | 注明估值方法 |
| 只写一面之词 | "ROCE 优秀，建议买入"忽略风险 | 同时列出风险和不确定性 |

### 2.3 避免 AI 幻觉

| 检查项 | 说明 |
|--------|------|
| 是否存在报告中有但数据来源中没有的指标 | AI 可能自行"补充"常见指标 |
| 是否存在"推测性"数据但未标注 | "预计营收增长 10%"→ 必须标注为预测 |
| 行业对比数据是否来自实际调用 | "行业平均 PE 为 XX"→ 确认是否从 skill 获取 |
| 公司描述是否有事实错误 | "主营业务"是否准确，是否张冠李戴 |

### 2.4 免责声明和局限性

检查报告是否包含：
- [ ] 明确的免责声明（本报告仅供参考，不构成投资建议）
- [ ] 数据截止日期
- [ ] 数据来源列表
- [ ] 可能影响分析的限制条件（如数据源覆盖范围）

## 第三部分：逐章审核要点

| 章节 | 数据准确性重点 | 观点客观性重点 |
|------|---------------|---------------|
| 一、公司概况 | 股票代码/名称/行业/市值是否正确 | 公司描述是否客观准确 |
| 二、近期行情 | 股价/涨跌幅/分位数与 skill 一致 | 行情分析是否仅描述事实 |
| 三、财务分析 | 营收/利润/ROE/负债逐行核对 | 趋势判断是否有数据支持 |
| 四、业务板块 | 业务数据是否准确 | 业务前景是否有合理依据 |
| 五、估值分析 | PE/PB/股息率与 skill 一致 | "低估/高估"判断是否有标准 |
| 六、风险因素 | 风险标注是否匹配 skill 输出 | 风险描述是否夸大/淡化 |
| 七、股东结构 | 持股比例/增减持是否准确 | 股东变化解读是否中立 |
| 八、综合评分 | 各维度得分求和是否正确 | 评分标准是否一致使用 |
| 九、投资建议 | 目标价计算是否有据可查 | 建议与评分是否逻辑自洽 |
| 十、结论 | 总结中的数据是否与报告一致 | 优劣势表述是否均衡 |

## 第四部分：修正流程

当发现数据错误或观点不客观时：

```
发现错误 → 追踪到来源 → 确认正确值 → 修正报告 → 重新核查
```

### 4.1 数据错误修正

1. 定位：哪个 Skill 的哪个字段填错了
2. 确认：重新调用对应 Skill 获取正确值
3. 修正：只在报告中修改，不修改原始数据
4. 联动检查：修正后检查所有关联章节是否也需要改

### 4.2 观点偏差修正

1. 确认偏差类型：无数据支撑 / 主观表述 / 建议与评分矛盾
2. 补充数据：如果观点正确但缺少数据支撑，补充数据
3. 调整措辞：将主观表述改为数据驱动表述
4. 一致性检查：调整后检查整篇报告的调性是否统一

### 4.3 重新审核

修正完成后，重点检查：
- [ ] 修正处是否正确
- [ ] 关联章节是否一致更新
- [ ] 整体评分/建议是否需要调整
- [ ] 数据来源标注是否更新
