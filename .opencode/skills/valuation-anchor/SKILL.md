# Valuation Anchor Method

**功能**: 估值锚点法 - 多维度交叉验证的买入点定价框架  
**数据源**: akshare + 内部计算逻辑

---

## 使用方式

**命令行运行**:
```bash
python .opencode/skills/valuation-anchor/scripts/main.py <股票代码>
```

### 示例

```bash
python .opencode/skills/valuation-anchor/scripts/main.py 600519   # 贵州茅台
python .opencode/skills/valuation-anchor/scripts/main.py 000895   # 双汇发展
```

## 方法概述

买入点 = 多个独立估值维度交叉验证后的价格区间。单一维度不够可靠，需要至少3个维度同时指向同一区间。

---

## 维度1：52周价格分布法

**原理**: 一年内的价格分布反映了市场对该股的全部定价共识。在中位数以下买入，统计上赢面更大。

**步骤**:
1. 获取 52周最高价（H）和最低价（L）
2. 计算中位数 M = (H + L) / 2
3. 计算 1/4 分位 Q1 = L + (H - L) * 0.25
4. 计算 3/4 分位 Q3 = L + (H - L) * 0.75

**定价规则**:

| 策略 | 价格区间 | 含义 |
|------|---------|------|
| 激进 | Q1 ~ M | 回调到中位数以下 |
| 稳健 | L + (M - L) * 0.2 ~ Q1 | 偏低1/4区间 |
| 理想 | L 附近 | 极端低价 |

---

## 维度2：PE 合理估值法

**原理**: 用当前每股收益（EPS）乘以合理PE区间，得到合理价格范围。

**步骤**:
1. 获取当前 EPS（最近12个月或最新年报）
2. 确定该行业的合理PE区间（参考同类公司PE均值和历史PE中位数）
3. 合理价格 = EPS * PE合理区间

**常见行业PE参考**:

| 行业类型 | 低估PE | 合理PE | 高估PE |
|---------|--------|--------|--------|
| 银行 | <5 | 5-7 | >8 |
| 公用事业 | <8 | 8-12 | >14 |
| 消费品（成熟） | <15 | 15-20 | >25 |
| 消费品（成长） | <20 | 20-35 | >40 |
| 科技/医药 | <25 | 25-40 | >50 |
| 周期股 | 看PB | — | — |

**定价规则**:

| 策略 | 价格区间 | 含义 |
|------|---------|------|
| 激进 | EPS * 合理PE下限 | 回到合理偏下 |
| 稳健 | EPS * (低估PE ~ 合理PE下限) | 明显偏低 |
| 理想 | EPS * 低估PE | 极端低估 |

---

## 维度3：股息率锚定法

**原理**: 对高分红股票，股息率是最硬的"确定性收益率"锚点。用目标股息率倒推目标买入价。

**步骤**:
1. 获取最近年度每股分红（DPS）
2. 设定目标股息率门槛：
   - 基准门槛：4%（跑赢银行理财）
   - 优秀门槛：5%（有吸引力的分红收益）
   - 极致门槛：6%（难得的底部机会）
3. 目标价格 = DPS / 目标股息率

**定价规则**:

| 策略 | 目标股息率 | 含义 |
|------|-----------|------|
| 激进 | >=4% | 略有分红优势 |
| 稳健 | >=5% | 明显的分红吸引力 |
| 理想 | >=6% | 难得的底部机会 |

**适用条件**: 仅适用于分红率稳定（>50%）、分红连续（>3年）的公司。

---

## 维度4：技术面验证

**原理**: 基本面和估值算出的价格区间，需要技术面验证是否实际有支撑。

**关键指标**:
- **历史支撑位**: 过去一年多次回踩后反弹的价格区域
- **均线支撑**: 60日/120日均线位置（长期均线支撑更强）
- **成交密集区**: 长期横盘整理的价格区间
- **缺口/前高**: 重要价格关口

**验证规则**:
- 估值算出的买入区间内有技术面支撑 → ✅ 增强信心
- 估值算出的买入区间内无技术面支撑 → ⚠️ 降低仓位或观望
- 技术面支撑位远高于估值买入区间 → 重新评估估值假设

---

## 交叉验证与最终定价

### 合成方法

将4个维度各自算出的价格区间叠加，找出**交集最大**的价格带：

```
维度1（价格分布）：给出区间 A
维度2（PE估值）  ：给出区间 B
维度3（股息率）  ：给出区间 C
维度4（技术面）  ：给出支撑位 S1, S2, S3

甜蜜点 = A ∩ B ∩ C ∩ (S1附近 或 S2附近)
```

### 输出要求

最终报告必须包含：
1. 每个维度的独立计算表（价格 → 对应指标值）
2. 四维交叉验证图（文字描述即可）
3. 明确标注哪些结论来自**客观数据**（如PE倒推），哪些来自**经验判断**（如PE合理区间本身）
4. 坦诚承认不确定性（"合理PE区间是经验值，不同分析师可能有不同判断"）

## 特殊情况处理

### 高成长股（营收增速>20%）
- PE估值法权重降低（成长股PE波动大）
- 增加 PEG 估值维度（PE / 增速 < 1 为低估）
- 股息率锚定法可能不适用（成长股通常低分红）

### 周期股（银行、钢铁、煤炭、猪企等）
- PE估值法不适用（周期股PE在景气周期高点反而低）
- 改用 PB 估值法（PB < 1 或接近历史最低 = 买入机会）
- 股息率锚定法依然适用

### 亏损股
- PE法不可用
- 改用 PS（市销率）或 PB 估值
- 重点关注收入增速、市场份额变化、扭亏预期

### 新股（上市<2年）
- 52周价格法参考价值有限
- 增加行业对标维度（同行业成熟公司估值对比）
- 提高风险权重
