# 行业分析框架

**目标**: 对A股行业板块进行多维度数据分析（排行/资金流/估值/成分股/轮动），支持指定行业深度分析
**Skill**: `industry-analysis`
**数据源**: Tushare Pro（申万行业 sw_daily）+ akshare（同花顺/东方财富/巨潮）替补

---

## 框架架构

```
industry-analysis/
├── SKILL.md                    # 总入口（本文件）
├── guides/                     # 分析指南目录
│   ├── 00-industry-name-list.md         # ⭐ 行业名称列表（先查后执行）⭐
│   ├── 01-industry-ranking.md           # 行业排行分析
│   ├── 02-industry-fundflow.md          # 行业资金流向分析
│   ├── 03-industry-valuation.md         # 行业估值分析
│   ├── 04-industry-rotation.md          # 行业轮动与综合判断
│   └── 05-single-industry-deep-dive.md  # 单个行业深度分析
└── core/src/skills/industry-analysis/
    ├── main.py                 # 统一入口（无参→全市场；有参→单行业深度）
    ├── collect_industry_spot.py     # 行业实时行情采集
    ├── collect_industry_fundflow.py # 行业资金流向采集
    ├── collect_industry_detail.py   # 行业估值 + 成分股采集
    └── collect_single_industry.py   # 单个行业深度采集 ⭐
```

## 分析流程

```
Step 1: 行业排行分析
  采集: python collect_industry_spot.py
  分析: → guides/01-industry-ranking.md
  产出: 涨幅榜/跌幅榜/成交额榜 Top 10 + 行业涨跌分布

Step 2: 行业资金流向
  采集: python collect_industry_fundflow.py
  分析: → guides/02-industry-fundflow.md
  产出: 主力净流入/流出榜 Top 10 + 全市场资金汇总

Step 3: 行业估值分析
  采集: python collect_industry_detail.py
  分析: → guides/03-industry-valuation.md
  产出: 行业PE/PB估值 + 高估值/低估值排行

Step 4: 行业轮动与综合
  采集: 前三步数据汇总
  分析: → guides/04-industry-rotation.md
  产出: 行业轮动判断 + 三表交叉验证 + 综合建议

单行业深度分析 ⭐
  先解析行业名：获取行业名称列表 → 匹配用户输入 → 不确认则让用户选择 → 再执行分析
  采集: python main.py <行业名>
  分析: → guides/05-single-industry-deep-dive.md
  产出: 指定行业的行情/成分股/资金流/估值四维数据
```

## 【必须遵守】单行业名称解析规则 ⭐

> **核心原则**：用户输入模糊行业名（如"家电"）后，**不允许直接运行代码**。
> 必须由你 **手动查阅行业名称列表**，列出所有匹配项让用户确认，确认后才能执行分析。

### 流程（严格执行，不可跳过）

```
用户输入模糊行业名
    ↓
你手动查阅 guides/00-industry-name-list.md
    ↓
你列出所有匹配项（不加推荐、不加判断）
    ↓
用户确认具体行业名称
    ↓
你用确认后的名称执行分析代码
```

### 执行步骤

**Step 1 —你查列表（禁止用代码查）**
-你打开 `guides/00-industry-name-list.md`
- 人工逐行查找包含用户输入关键词的行业名称
- **禁止**: 调用任何 Python/命令行代码来搜索
- **禁止**: 用代码模糊匹配
- **必须**:你用 grep/搜索工具或直接阅读文件来找

**Step 2 —你列出全部匹配项**
- 将所有匹配的行业名称完整列出
- **不加推荐**（不写"推荐"、"最相关"等主观判断）
- **不加分类**（不写"家电相关细分"等分组）
- 按列表原样呈现给用户

**Step 3 — 用户确认**
- 用户从列表中选定一个具体行业名称
- 用户也可以说"算了"放弃分析

**Step 4 —你执行分析**
- 用用户确认的**准确行业名称**调用分析代码
- 命令: `python main.py "<准确名称>"`

### 示例（以本次实际对话为准）

```
用户: 家电

AI: 查阅 guides/00-industry-name-list.md，含"家电"的行业有:
   - 其他黑色家电

   - 家电零部件Ⅲ

   - 白色家电

   请选择要分析哪个？

用户: 家用电器

AI: 执行 python main.py "家用电器"
    → [输出分析报告]
```

### 变更历史对比

| 步骤 | 旧规则（已废弃） | 新规则（强制执行） |
|------|-----------------|-------------------|
| 匹配方式 | 代码 `str.contains()` 自动匹配 |你手动查列表文件 |
| 唯一匹配 | 直接使用，不通知用户 | 仍然列出让用户确认 |
| 多个匹配 | 列出 + 推荐 | 只列出，不加推荐 |
| 执行时机 | 匹配完直接执行 | 用户确认后才执行 |

## 三表交叉验证体系

| 维度 | 数据来源脚本 | 核心字段 |
|------|-------------|----------|
| 价格维度（谁在涨/跌） | `collect_industry_spot.py` | change_pct / volume / leader_change |
| 资金维度（钱去了哪） | `collect_industry_fundflow.py` | main_net_inflow / inflow_sectors_count |
| 估值维度（贵还是便宜） | `collect_industry_detail.py` | pe / pb / pe_median / pb_median |

### 信号组合

| 组合 | 含义 | 置信度 |
|------|------|--------|
| ↑价格 + ↑资金 + 合理估值 | 健康上涨，趋势可持续 | ★★★★★ |
| ↑价格 + ↓资金 + 高估值 | 诱多/赶顶，警惕回调 | ★★★★☆ |
| ↓价格 + ↓资金 + 高估值 | 估值回归，继续回避 | ★★★★☆ |
| ↓价格 + ↑资金 + 低估值 | 资金抄底，关注反转 | ★★★☆☆ |

## 行业分类体系

| 类别 | 典型行业 | 风险偏好代表 |
|------|----------|-------------|
| 🚀 进攻型 | 半导体/新能源/计算机/TMT | 高风险偏好（牛市） |
| ⚖️ 均衡型 | 机械/化工/汽车/消费电子 | 中等风险偏好 |
| 🛡️ 防御型 | 医药/食品饮料/公用事业 | 低风险偏好（熊市） |
| 🔄 周期型 | 银行/地产/煤炭/钢铁 | 特殊周期驱动 |

1. **数据实时性**：行业行情数据为实时数据，不同时点差异大
2. **Tushare Pro 优先**：行业实时行情（含PE/PB）优先使用 Tushare sw_daily 接口（439个申万行业）。若 Tushare 不可用，自动降级到 akshare 同花顺（49个行业）/巨潮。
3. **先解析行业名，再执行分析**：用户输入模糊行业名（如"家电"）后，必须先获取行业列表、匹配准确名称、确认后再执行。**不得直接运行代码**。
4. **东方财富接口不稳定**：`stock_sector_fund_flow_rank`（资金流）和 `stock_board_industry_cons_em`（成分股）依赖东方财富接口，偶有超时或连接断开，遇此情况稍后重试即可。同花顺行业板块接口 `stock_sector_spot` 较稳定。
5. **巨潮接口不稳定**：`stock_industry_pe_ratio_cninfo`（行业PE/PB估值）依赖巨潮资讯接口，偶有超时，重试即可
6. **行业分类差异**：不同数据源的行业分类标准不同，Tushare 申万 / 同花顺 / 东方财富 / 巨潮的行业划分有差异。Tushare 使用申万分类（439个三级行业），akshare 使用同花顺分类（49个行业）。
7. **估值数据更新**：Tushare sw_daily 含当日PE/PB数据；巨潮行业PE数据更新频率为日频，非实时
8. **综合分析**：单一维度不构成判断，需三表交叉验证
