## 三、分析维度

### 维度1：回购进度

```
完成比例 = 已回购金额 / 计划回购金额 × 100%
```

### 维度2：回购目的

| 目的 | 信号强度 | 说明 |
|------|----------|------|
| 回购并注销 | ★★★★★ | 提升每股价值 |
| 股权激励 | ★★★☆☆ | 绑定利益 |
| 员工持股 | ★★★☆☆ | 激励员工 |
| 市值管理 | ★★☆☆☆ | 维护股价 |

### 维度3：管理层信心

| 回购特征 | 信号强度 | 说明 |
|----------|----------|------|
| 溢价回购 | ★★★★★ | 强烈看好 |
| 大额回购 | ★★★★☆ | 资金支持 |
| 持续回购 | ★★★☆☆ | 长期规划 |
| 按计划完成 | ★★★☆☆ | 执行力 |

## 三、信号判定规则

### 回购信号强度

| 信号 | 条件 | 置信度 | 含义 |
|------|------|--------|------|
| 超额回购 | >100% | ★★★★★ | 强烈看好 |
| 溢价回购 | >20% | ★★★★★ | 估值偏低 |
| 大额回购 | >50亿 | ★★★★☆ | 资金充裕 |
| 按时完成 | 100% | ★★★☆☆ | 执行力强 |
| 正在进行 | >0% | ★★★ | 支撑股价 |
| 未开始 | 0% | - | 待执行 |
| 取消/变更 | <50% | ★★ | 可能放弃 |

### 回购与股价关系

```
回购期间    →  股价支撑
回购完成   →  每股价值提升
注销完成   →  永久性股权减少
```

## 四、分析流程

### Step 1：获取数据

```python
from core.shareholder import get_repurchase_data

repurchase = get_repurchase_data("000651")
# returns:
# {
#     "repurchases": [...],
#     "summary": {
#         "是否正在进行回购": "是",
#         "回购次数": 5
#     }
# }
```

### Step 2：信号判定

```python
for rpt in repurchase["repurchases"]:
    pct = rpt["完成比例"]
    status = rpt["执行状态"]
    
    if pct > 100:
        signal = "超额完成"
    elif status == "已完成" and pct >= 100:
        signal = "完成"
    elif status == "进行中":
        signal = "进行中"
    else:
        signal = "待执行"
```

### Step 3：生成信号

```python
if signal in ["超额完成", "完成"]:
    confidence = "Strong"
    action = "支撑股价"
elif signal == "进行中":
    confidence = "Medium"
    action = "持有观望"
else:
    confidence = "Weak"
    action = "观察"
```
