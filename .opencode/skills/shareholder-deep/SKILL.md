# 股东深度分析框架

**目标**: 对A股上市公司股东结构进行多维度、智能化的深度分析
**Skill**: `shareholder-deep`
**数据源**: Tushare Pro + AKShare

## 分析流程

```
输入：股票代码(如 000651)
    │
    ├──> Step 1: 基本信息
    │   获取：十大股东 / 类型分布 / 历史变化
    │   → guide/01-shareholder-structure.md
    │
    ├──> Step 2: 变化信号
    │   获取：增减持统计 / 抛售风险
    │   → guide/02-shareholding-change-signals.md
    │
    ├──> Step 3: 风险排查
    │   获取：股权质押 / 股份回购
    │   → guide/03-equity-pledge.md
    │   → guide/04-share-buyback.md
    │
    ├──> Step 4: 信心信号
    │   获取：管理层增持 / 回购注销
    │   → guide/05-management-share-increase.md
    │
    ├──> Step 5: 关联分析
    │   股东 → 财务质量    → guide/06-financial-correlation.md
    │   股东 → 估值水平    → guide/07-valuation-correlation.md
    │
    └──> Step 6: 行动建议
        综合判断 → guide/08-action-recommendations.md
```

## ✅ 分析工作流 Todo 清单

> 按顺序执行，每完成一步勾选一项。

### ▷ 数据采集

- [ ] 调用 Tushare `top10_floatholders` — 获取十大流通股东数据
- [ ] 调用 Tushare `top10_floatholders` 历史 — 获取股东变化数据
- [ ] 调用 AKShare `stock_pledge_ratio_em` — 获取股权质押数据
- [ ] 调用 AKShare `stock_repurchase_em` — 获取股份回购数据
- [ ] 调用 Tushare `holder_trade` — 获取管理层增减持数据
- [ ] 调用 Tushare 财务指标 — 获取财务关联数据（ROE/负债率）
- [ ] 调用 Tushare 估值指标 — 获取估值关联数据（PE/PB）

### ▷ 分析推理

- [ ] **Step 1 股东结构** — 集中度计算、控股类型判断、机构持股分析
- [ ] **Step 2 增减持信号** — 净增持/减持统计、大幅变化识别
- [ ] **Step 3 股权质押** — 质押比例计算、风险等级判定
- [ ] **Step 4 股份回购** — 回购次数/金额统计、回购目的分析
- [ ] **Step 5 管理层增持** — 增持频次/金额、关键人物识别
- [ ] **Step 6 财务关联** — 股东结构与财务质量交叉分析
- [ ] **Step 7 估值关联** — 股东结构与估值水平交叉分析

### ▷ 综合输出

- [ ] **Step 8 综合行动** — 加权评分（A-E 评级）
- [ ] 推导链条完整 — 每个步骤有【数据获取→计算推理→判断→结论】结构
- [ ] 信号方向标注 — 每个维度标注 positive/negative/neutral
- [ ] 置信度标注 — 每个结论附置信度（★评级）
- [ ] 行动建议明确 — 买入/持有/观望/回避，有对应数据支撑

## 【核心要求】推理过程必须严格展示
**每个分析步骤必须包含完整的推理链条，不能只输出结论！**

### 推理过程标准格式

```
【分析Step X: 维度名称】
[数据获取]
  1. 调用XX接口获取XX数据...
  2. 获取到XX条记录
[计算推理]
  1. 计算指标A = ...
  2. 计算指标B = ...
  3. 与阈值对比判断：...

[判断]
  → 指标A结果：XX
  → 指标B结果：XX  
  → 综合判断：XX

[结论]
  - 信号方向：positive/negative/neutral
  - 置信度：★★★★☆
  - 行动建议：买入/持有/观望/回避
```

### 示例输出

```bash
【分析Step 1: 股东结构】
[数据获取]
  1. 调用get_top_circulating_holders获取十大流通股东...
  2. 获取到10条记录
[计算推理]
  1. 计算股权集中度 = sum(占流通股比例) = 79.9%
  2. 与阈值对比：79.9% > 70% → 高度集中
  3. 识别第一大股东：罗特克斯，持股70.3%
  4. 判断控股类型：70.3% > 50% → 绝对控股

[判断]
  → 集中度结果：高度集中
  → 控股类型：绝对控股
  → 信号强度：⭐★★★★

[结论]
  - 信号方向：positive
  - 置信度：★★★★☆
  - 行动建议：持有
【分析Step 2: 增减持信号】
[数据获取]
  1. 调用get_holder_changes获取近5期变化...
  2. 获取到5条变动记录
[计算推理]
  1. 统计增持次数 = 6
  2. 统计减持次数 = 3
  3. 计算净变化 = 6 - 3 = 3 (净增持)
  4. 判断趋势：增持 > 减持 → 净增持趋势

[判断]
  → 趋势：净增持
  → 大幅变化：华泰ETF增持11.1%，华泰柏瑞减持1.0%
  → 信号：positive

[结论]
  - 信号方向：positive
  - 置信度：★★★★☆
  - 行动建议：关注
```

## 信号强度体系

### 置信度等级
| 等级 | 符号 | 含义 | 使用场景 |
|------|------|------|----------|
| 5星 | ⭐⭐⭐⭐⭐ | 确定性信号 | 回购注销、管理层增持 |
| 4星 | ⭐⭐⭐⭐☆ | 强信号 | 控股股东增持、国家队进入 |
| 3星 | ⭐⭐⭐☆☆ | 中性信号 | 机构进入、经销商增持 |
| 2星 | ⭐⭐☆☆☆ | 弱信号 | 股权激励、回购用于激励 |
| 1星 | ⭐☆☆☆☆ | 风险信号 | 大股东减持、连续抛售 |

### 时效性修正
| 时效 | 修正系数 | 说明 |
|------|----------|------|
| 当期 | ×1.0 | 本季度数据 |
| 1年内 | ×0.8 | 略有滞后 |
| 2年前 | ×0.5 | 历史数据 |

### 最终置信度 = 信号强度 × 时效系数

## 注意事项

1. **数据滞后**：股东数据季度更新，注意时效
2. **代持风险**：表面股东可能是代持
3. **BVI公司**：离岸公司看不清真实控制人
4. **综合判断**：单一信号不构成投资建议
